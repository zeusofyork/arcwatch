Get your API keys

You need an API key from each provider you want to track. Keys require read access to usage data — no write permissions needed.

Anthropic
console.anthropic.com → API Keys
Key format: sk-ant-api03-…
OpenAI
platform.openai.com → API keys
Key format: sk-…
2

Add the key in Settings

Go to Settings → LLM APIs and click Add Key. Select the provider, give it a label (e.g. "Production"), and paste the key.

Keys are encrypted with AES-256 (Fernet) before storage and never displayed in plaintext. You can add multiple keys per provider.
Go to Settings → LLM APIs
3

Sync data

Click Sync Now next to your provider to pull the last 32 days of usage immediately. After that, data syncs automatically every hour.

Once synced, the LLM APIs dashboard will show spend totals, projected month-end cost, and a per-model breakdown.

FAQ

How often does data sync?
Automatically every hour. You can also click Sync Now for an immediate refresh.
What date range is covered?
Each sync fetches the last 32 days. The dashboard shows the current calendar month. The daily chart shows the last 30 days.
Why is the dashboard showing no data?
Either no sync has run yet (click Sync Now), or your account has no usage in the current month. Check the Last Synced column in Settings → LLM APIs.
What permissions are needed to add a provider key?
Owner or Admin role. Viewers can see the dashboard but cannot add, delete, or sync providers.
What is "Cache Hit Rate"?
Anthropic only. The percentage of input tokens served from prompt cache (cache_read_tokens / (input_tokens + cache_read_tokens)). Higher is cheaper — cached tokens cost ~10% of full input tokens.
Can I add multiple keys for the same provider?
Yes. Each key syncs independently and usage is aggregated per organization. Useful if you have separate keys for staging and production.
